# Ticketverwaltung – GitHub-Pages-Onefile

Diese Version ist für das Hochladen per Handy optimiert.

## Dateien

- `index.html` – enthält HTML, CSS und JavaScript vollständig zusammengeführt.
- `404.html` – identische Fallback-Datei, damit geteilte Links bei GitHub Pages sauber geöffnet werden können.
- `.nojekyll` – deaktiviert Jekyll-Verarbeitung für GitHub Pages.

## Upload auf GitHub

1. Repository `ticket-test` öffnen.
2. Die Dateien aus dieser ZIP in den Repository-Root hochladen.
3. In `Settings → Pages` als Quelle `Deploy from a branch` wählen.
4. Branch `main` und Ordner `/root` auswählen.
5. Die Seite ist danach unter `https://wickerm4n.github.io/ticket-test/` erreichbar.

## Hinweis zur Share-Funktion

Diese reine GitHub-Pages-Version ist statisch. Read-only-Snapshot-Links funktionieren direkt.
Editierbare Live-Share-Links mit gemeinsamem Datenstand benötigen weiterhin ein separates Backend.
